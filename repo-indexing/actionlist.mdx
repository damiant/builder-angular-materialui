# ActionList Components

## Overview

### Purpose

Use this component to generate action list functionality

### When to Use

ActionList components are ideal for creating interactive lists where users can trigger actions on list items. Use them when you need to:

- **Interactive content lists** - Create lists where users can click, tap, or interact with individual items to perform actions
- **Navigation menus** - Build clickable navigation lists that route users to different sections or pages
- **Settings and configuration interfaces** - Provide interactive lists for user preferences, settings, or configuration options
- **Command palettes** - Implement lists of available actions or commands that users can select from
- **File browsers** - Create lists of files, folders, or documents that users can open, select, or manipulate
- **Contact lists** - Build interactive contact or user lists with click-to-call, message, or profile actions
- **Menu-like interfaces** - Implement dropdown menus, context menus, or sidebar navigation with actionable items
- **Dashboard widgets** - Create interactive list components for dashboards where items represent actionable data

### Architecture

**No Coupling**: The ActionList component is a standalone component that operates independently without requiring other components or shared context. It provides enhanced list functionality with built-in interaction capabilities and can be used in isolation or combined with other components as needed.

### Component Interdependencies

The ActionList component group has no interdependencies - MatActionList is a self-contained component that:

- Requires no parent components or containers
- Does not depend on shared context or state management
- Can be used independently or as a building block for interactive interfaces
- Provides enhanced interaction capabilities beyond basic MatList functionality
- Integrates seamlessly with Angular routing and event handling systems

## Components

### MatActionList {Standalone Component}

An interactive list component that provides built-in click handling, keyboard navigation, and accessibility features for actionable list items.

**Purpose:** Provides the foundation for creating interactive lists where each item can trigger actions, navigate to routes, or perform user-defined behaviors with proper Material Design styling and accessibility support.

**Interface:**

```typescript
interface MatActionListProps {
  children: React.ReactNode; // List content, typically mat-list-item elements with actions
  disableRipple?: boolean; // Disables ripple effects on list items (default: false)
  disabled?: boolean; // Disables the entire list and all item interactions (default: false)
  id?: string; // Unique identifier for the list
  class?: string; // CSS classes for additional styling
  role?: string; // ARIA role for accessibility (default: 'list')
}
```

**Usage Example:**

```tsx
import { MatActionList } from "@angular/material/list";

function BasicActionListExample() {
  return (
    <mat-action-list>
      <mat-list-item (click)="handleItemClick('item1')">
        <mat-icon matListItemIcon>settings</mat-icon>
        <span matListItemTitle>Settings</span>
        <span matListItemLine>Configure application preferences</span>
      </mat-list-item>
      <mat-list-item (click)="handleItemClick('item2')">
        <mat-icon matListItemIcon>account_circle</mat-icon>
        <span matListItemTitle>Profile</span>
        <span matListItemLine>Manage your account information</span>
      </mat-list-item>
      <mat-list-item (click)="handleItemClick('item3')">
        <mat-icon matListItemIcon>logout</mat-icon>
        <span matListItemTitle>Logout</span>
        <span matListItemLine>Sign out of your account</span>
      </mat-list-item>

  );
}
```

### Advanced Usage

```tsx
// Complex action list with routing, conditional actions, and rich content
@Component({
  template: `
    <mat-action-list
      [disableRipple]="disableInteractions"
      [disabled]="isLoading"
    >
      <mat-list-item
        *ngFor="let action of availableActions; trackBy: trackByAction"
        (click)="executeAction(action)"
        [class.disabled]="action.disabled"
        [attr.aria-label]="action.ariaLabel"
      >
        <mat-icon
          matListItemIcon
          [color]="action.iconColor"
          [class.disabled-icon]="action.disabled"
        >
          {{ action.icon }}
        </mat-icon>

        <div matListItemTitle>{{ action.title }}</div>
        <div matListItemLine>{{ action.description }}</div>

        <mat-badge
          *ngIf="action.badge"
          matListItemMeta
          [content]="action.badge.content"
          [color]="action.badge.color"
        >
        </mat-badge>

        <mat-icon *ngIf="action.showChevron" matListItemMeta>
          chevron_right
        </mat-icon>
      </mat-list-item>

      <mat-divider *ngIf="showDividers"></mat-divider>

      <mat-list-item
        *ngFor="let secondaryAction of secondaryActions"
        (click)="executeSecondaryAction(secondaryAction)"
        [routerLink]="secondaryAction.route"
      >
        <mat-icon matListItemIcon>{{ secondaryAction.icon }}</mat-icon>
        <div matListItemTitle>{{ secondaryAction.title }}</div>
      </mat-list-item>
    </mat-action-list>
  `,
})
export class AdvancedActionListExample {
  disableInteractions = false;
  isLoading = false;
  showDividers = true;

  availableActions = [
    {
      id: "notifications",
      title: "Notifications",
      description: "Manage your notification preferences",
      icon: "notifications",
      iconColor: "primary",
      disabled: false,
      badge: { content: "3", color: "warn" },
      showChevron: true,
      ariaLabel: "Open notification settings",
    },
    {
      id: "privacy",
      title: "Privacy & Security",
      description: "Control your data and security settings",
      icon: "security",
      iconColor: "accent",
      disabled: false,
      showChevron: true,
      ariaLabel: "Open privacy settings",
    },
    {
      id: "billing",
      title: "Billing",
      description: "Manage subscription and payment methods",
      icon: "payment",
      iconColor: "warn",
      disabled: true,
      showChevron: false,
      ariaLabel: "Billing settings (currently unavailable)",
    },
  ];

  secondaryActions = [
    {
      id: "about",
      title: "About",
      icon: "info",
      route: "/about",
    },
    {
      id: "feedback",
      title: "Send Feedback",
      icon: "feedback",
      route: "/feedback",
    },
  ];

  executeAction(action: any): void {
    if (action.disabled) return;

    switch (action.id) {
      case "notifications":
        this.router.navigate(["/settings/notifications"]);
        break;
      case "privacy":
        this.router.navigate(["/settings/privacy"]);
        break;
      default:
        console.log(`Executing action: ${action.id}`);
    }
  }

  executeSecondaryAction(action: any): void {
    console.log(`Secondary action: ${action.id}`);
  }

  trackByAction(index: number, action: any): string {
    return action.id;
  }
}
```

## Best Practices

### Usage Guidelines

- **Use for interactive content** - MatActionList is specifically designed for lists where items trigger actions, unlike MatList which is for static content display
- **Implement proper click handlers** - Always provide meaningful click handlers for list items to ensure interactive functionality works as expected
- **Consider keyboard accessibility** - Ensure that all actions can be triggered via keyboard navigation using Enter or Space keys
- **Provide visual feedback** - Use ripple effects and hover states to indicate interactive elements and provide immediate user feedback

### Accessibility

- **ARIA labels and roles** - MatActionList automatically provides proper ARIA attributes, but add custom aria-label attributes for complex actions
- **Keyboard navigation** - Built-in support for arrow key navigation and Enter/Space activation following ARIA best practices
- **Screen reader support** - Action descriptions and states are automatically announced to assistive technologies
- **Focus management** - Proper focus indication and restoration when navigating between list items and performing actions

### Performance

- **Use trackBy functions** - When rendering dynamic action lists with \*ngFor, provide trackBy functions to optimize change detection
- **Minimize action handler complexity** - Keep click handlers lightweight and delegate heavy operations to services or use async operations
- **Optimize large lists** - For action lists with many items, consider virtual scrolling or pagination to maintain performance
- **Event delegation** - Use event delegation patterns for similar actions to reduce memory usage and improve performance
