# MiniFabButton Components

## Overview

### Purpose

Use this component to generate mini floating action button functionality. The MiniFabButton component provides Material Design styled mini floating action buttons (Mini FABs) that enable users to access secondary actions in an application. Mini FABs are smaller, compact circular buttons that float above the UI and represent secondary or supporting actions on a screen, following Material Design 3 specifications for floating elements with proper elevation, accessibility features, and space-efficient design.

### When to Use

- **Secondary Actions**: When you need to provide access to secondary actions that support the primary workflow without competing for attention
- **Space-Constrained Interfaces**: In compact layouts where full-size FABs would be too prominent or take up excessive space
- **Toolbar Integration**: For action buttons within toolbars, navigation bars, or compact control panels
- **Multi-Action Interfaces**: When providing multiple quick actions in a grouped or list format without overwhelming the UI
- **Edit Mode Actions**: For inline editing controls like save, cancel, or delete actions within content areas
- **Navigation Controls**: For compact navigation elements like next/previous, up/down, or directional controls
- **Quick Toggle Actions**: For buttons that toggle states or settings in compact interfaces
- **Mobile Optimization**: Particularly effective on mobile devices where screen space is limited but actions need to remain accessible
- **Supporting Workflows**: When complementing primary actions with related but less prominent secondary operations
- **Dense Information Displays**: In data-heavy interfaces where actions need to be available without disrupting content density

### Architecture

**No Coupling**: The MiniFabButton component is a standalone component that operates independently without requiring other components. It provides:

- **Self-contained functionality**: Complete mini floating action button implementation with all necessary features built-in
- **Flexible positioning**: Can be positioned anywhere in the UI without dependencies on parent layout components
- **Consistent elevation**: Maintains proper Material Design elevation and shadow effects regardless of context
- **Theme integration**: Automatically inherits from the Material Design theme system while maintaining mini FAB-specific styling
- **Compact footprint**: Optimized for smaller spaces while maintaining touch accessibility and visual clarity

## Components

### MatMiniFabButton {Standalone Component}

A compact variant of the floating action button designed for secondary actions or when space is constrained, providing all the functionality of a standard FAB in a smaller form factor.

**Purpose:** Creates compact floating action buttons with reduced size for secondary actions, toolbar integration, or space-constrained interfaces while maintaining Material Design styling and accessibility.

**Interface:**

```typescript
interface MatMiniFabButtonProps {
  // Appearance and Styling
  color?: "primary" | "accent" | "warn" | string; // Theme color for the button

  // Interaction States
  disabled?: boolean; // Whether the button is disabled
  disableRipple?: boolean; // Whether to disable the ripple effect

  // Interactive Behavior
  disabledInteractive?: boolean; // Allows interaction even when disabled (for accessibility)

  // Accessibility
  ariaDisabled?: boolean; // Manual control over aria-disabled state
  ariaLabel?: string; // Required accessible label for screen readers

  // Focus and Navigation
  tabIndex?: number; // Tab order for keyboard navigation

  // Events (inherited from HTMLButtonElement)
  click?: () => void; // Click event handler
  focus?: () => void; // Focus event handler
  blur?: () => void; // Blur event handler
  mouseenter?: () => void; // Mouse enter event handler
  mouseleave?: () => void; // Mouse leave event handler
}
```

### Advanced Mini FAB Implementation

```typescript
import { MatButtonModule } from "@angular/material/button";
import { MatIconModule } from "@angular/material/icon";
import { MatTooltipModule } from "@angular/material/tooltip";
import { MatProgressSpinnerModule } from "@angular/material/progress-spinner";

@Component({

      <div class="state-fabs">

          mat-mini-fab

        >

          <mat-icon *ngIf="!isLoading">save</mat-icon>

        <button
          mat-mini-fab
          *ngIf="hasUnsavedChanges"

          aria-label="Discard changes"

          [attr.aria-label]="isBookmarked ? 'Remove bookmark' : 'Add bookmark'"
          [matTooltip]="isBookmarked ? 'Remove bookmark' : 'Add bookmark'"
        >
          <mat-icon>{{
            isBookmarked ? "bookmark" : "bookmark_border"
          }}</mat-icon>
        </button>
      </div>

      <!-- Toolbar with mini FABs -->
      <div class="mini-fab-toolbar" role="toolbar" aria-label="Content actions">

          [disabled]="!action.enabled"
          [color]="action.color"
          (click)="executeAction(action)"
          [attr.aria-label]="action.label"
          [matTooltip]="action.tooltip"
          class="toolbar-mini-fab"
        >
          <mat-icon>{{ action.icon }}</mat-icon>
        </button>
      </div>

      <!-- Responsive mini FAB group -->
      <div class="responsive-mini-fabs">
        <ng-container *ngFor="let fab of responsiveFabs">
          <button
            mat-mini-fab
            *ngIf="shouldShowFab(fab)"
            [color]="fab.color"
            (click)="handleFabAction(fab)"
            [attr.aria-label]="fab.label"
            [matTooltip]="fab.tooltip"
            class="responsive-mini-fab"
          >
            <mat-icon>{{ fab.icon }}</mat-icon>
          </button>
        </ng-container>
      </div>

      <!-- Context-sensitive mini FABs -->
      <div class="context-fabs" *ngIf="selectedItems.length > 0">
        <span class="selection-count">{{ selectedItems.length }} selected</span>

        <button
          mat-mini-fab
          (click)="bulkEdit()"
          aria-label="Edit selected items"
          matTooltip="Edit selected items"
          color="primary"
        >
          <mat-icon>edit</mat-icon>
        </button>

        <button
          mat-mini-fab
          (click)="bulkDelete()"
          aria-label="Delete selected items"
          matTooltip="Delete selected items"
          color="warn"
        >
          <mat-icon>delete</mat-icon>
        </button>

        <button
          mat-mini-fab
          (click)="clearSelection()"
          aria-label="Clear selection"
          matTooltip="Clear selection"
        >
          <mat-icon>clear</mat-icon>
        </button>
      </div>
    </div>
  `,
  styles: [

        padding: 16px;

      }

        display: flex;
        gap: 8px;
        margin-bottom: 24px;

        display: flex;
        gap: 8px;
        flex-wrap: wrap;
        margin-bottom: 24px;
      }

      .context-fabs {
        display: flex;
        align-items: center;
        gap: 8px;

      }

        color: #1976d2;
        margin-right: 8px;
      }

      @media (max-width: 600px) {
        .mini-fab-toolbar {
          flex-wrap: wrap;
        }

        .responsive-mini-fabs {
          justify-content: center;
        }

        .context-fabs {
          flex-wrap: wrap;
          justify-content: center;
        }
      }
    `,
  ],
  imports: [
    MatButtonModule,
    MatIconModule,
    MatTooltipModule,
    MatProgressSpinnerModule,
  ],
})
export class AdvancedMiniFabComponent {
  isLoading = false;
  hasUnsavedChanges = true;
  isBookmarked = false;
  selectedItems: any[] = [];

  toolbarActions = [
    {
      id: 1,
      icon: "copy",
      label: "Copy",
      tooltip: "Copy content",
      enabled: true,
      color: "primary",
    },
    {
      id: 2,
      icon: "share",
      label: "Share",
      tooltip: "Share content",
      enabled: true,
      color: "accent",
    },
    {
      id: 3,
      icon: "download",
      label: "Download",
      tooltip: "Download content",
      enabled: false,
      color: "",
    },
    {
      id: 4,
      icon: "print",
      label: "Print",
      tooltip: "Print content",
      enabled: true,
      color: "",
    },
  ];

  responsiveFabs = [
    {
      id: "edit",
      icon: "edit",
      label: "Edit",
      tooltip: "Edit item",
      color: "primary",
      showOnMobile: true,
      showOnDesktop: true,
    },
    {
      id: "info",
      icon: "info",
      label: "Info",
      tooltip: "View information",
      color: "accent",
      showOnMobile: false,
      showOnDesktop: true,
    },
    {
      id: "help",
      icon: "help",
      label: "Help",
      tooltip: "Get help",
      color: "",
      showOnMobile: true,
      showOnDesktop: false,
    },
  ];

  async performAsyncAction(): Promise<void> {
    this.isLoading = true;
    try {
      // Simulate async operation
      await new Promise((resolve) => setTimeout(resolve, 2000));
      this.hasUnsavedChanges = false;
      console.log("Action completed successfully");
    } catch (error) {
      console.error("Action failed:", error);
    } finally {
      this.isLoading = false;
    }
  }

  discardChanges(): void {
    this.hasUnsavedChanges = false;
    console.log("Changes discarded");
  }

  toggleBookmark(): void {
    this.isBookmarked = !this.isBookmarked;
    console.log("Bookmark toggled:", this.isBookmarked);
  }

  executeAction(action: any): void {
    if (action.enabled) {
      console.log("Executing action:", action.label);
    }
  }

  trackByActionId(index: number, action: any): number {
    return action.id;
  }

  shouldShowFab(fab: any): boolean {
    const isMobile = window.innerWidth <= 600;
    return isMobile ? fab.showOnMobile : fab.showOnDesktop;
  }

  handleFabAction(fab: any): void {
    console.log("FAB action:", fab.label);
  }

  bulkEdit(): void {
    console.log("Bulk editing", this.selectedItems.length, "items");
  }

  bulkDelete(): void {
    console.log("Bulk deleting", this.selectedItems.length, "items");
    this.selectedItems = [];
  }

  clearSelection(): void {
    this.selectedItems = [];
    console.log("Selection cleared");
  }
}
```

## Best Practices

### Usage Guidelines

- **Secondary action prioritization**: Use mini FABs for secondary actions that support the primary workflow without competing for user attention
- **Space-efficient grouping**: Group related mini FABs logically with consistent spacing (8-12px gaps) to maintain visual clarity
- **Consistent sizing**: Maintain the standard mini FAB size (40dp) to ensure accessibility and visual consistency across the interface
- **Appropriate color usage**: Use theme colors purposefully - primary for positive actions, warn for destructive actions, accent for special features
- **Clear visual hierarchy**: Position mini FABs to support, not compete with, primary actions and content flow

### Accessibility

- **Descriptive aria-labels**: Provide clear, context-specific aria-label attributes since mini FABs rely heavily on iconography
- **Keyboard navigation**: Ensure all mini FABs are reachable via Tab navigation and activatable with Enter or Space keys
- **Touch target compliance**: Maintain minimum 44dp touch targets with adequate spacing to prevent accidental activation
- **Screen reader support**: Use live regions to announce state changes and action results for dynamic mini FABs
- **Focus management**: Provide clear focus indicators that meet WCAG contrast requirements and are visible against all backgrounds

### Performance

- **Icon optimization**: Use Material Icons efficiently and consider icon loading strategies for large sets of mini FABs
- **Event delegation**: For lists with many mini FABs, consider event delegation patterns to optimize memory usage
- **Ripple performance**: Monitor ripple effects in dense layouts and consider disabling for performance-critical scenarios
- **State management**: Optimize state updates for dynamic mini FABs to prevent unnecessary re-renders
- **Animation efficiency**: Use CSS transforms for smooth transitions and avoid animating layout-affecting properties
