# GridList Components

## Overview

### Purpose

Use this component group to generate grid list functionality with tile support. The GridList component group provides a comprehensive solution for displaying content in a responsive grid layout with intelligent tile positioning, supporting variable tile sizes and automatic layout calculations.

### When to Use

- **Image Galleries**: Create responsive photo galleries with varying image sizes and aspect ratios
- **Product Catalogs**: Display products, services, or items in an organized grid format
- **Dashboard Widgets**: Arrange dashboard components in a flexible grid layout with different sizes
- **Media Libraries**: Organize videos, documents, or other media in a visually appealing grid
- **Card Layouts**: Present information cards with consistent spacing and alignment
- **Portfolio Displays**: Showcase work samples, projects, or case studies in a professional grid
- **Content Previews**: Display article previews, blog posts, or content summaries in grid format

### Architecture

**Critical/Strong Coupling**: The GridList component group requires strict hierarchical relationships and shared context between components. All tiles must be direct children of the grid list container to ensure proper positioning and responsive behavior.

### Component Interdependencies

The GridList components form a tightly coupled system with specific requirements:

- **MatGridList**: Root container that manages grid layout, column calculations, and tile positioning algorithm
- **MatGridTile**: Individual tile components that specify their size requirements (rowspan, colspan)
- **Tile Layout Algorithm**: Automatic positioning system that places tiles optimally based on available space
- **Responsive Calculations**: Dynamic column and row sizing based on container dimensions and configuration
- **Shared Context**: Grid configuration (columns, gutter size, row height) affects all child tiles

## Components

### MatGridList {Primary Component}

**Purpose:** The root container component that manages the grid layout, calculates tile positions, and provides responsive grid behavior with configurable columns, row heights, and gutters.

**Interface:**

```typescript
interface MatGridListProps {
  children: React.ReactNode; // Required MatGridTile components
  cols: number; // Number of columns in the grid (required)
  gutterSize?: string; // Space between tiles (default: '1px')
  rowHeight?: string | number; // Height configuration for rows (default: '1:1')
}

// Row height options:
// - Fixed height: '100px', '5rem'
// - Aspect ratio: '4:3', '16:9', '1:1'
// - Fit mode: 'fit' (divides available height by number of rows)
```

**Dependencies:** None (root component)
**Provides:** Grid layout context, positioning algorithm, responsive calculations for all child tiles

**Usage Example:**

```typescript
import { MatGridListModule } from "@angular/material/grid-list";

@Component({
  template: `
    <mat-grid-list [cols]="4" gutterSize="10px" rowHeight="2:1">
      <mat-grid-tile>Content 1</mat-grid-tile>
      <mat-grid-tile>Content 2</mat-grid-tile>
      <mat-grid-tile [colspan]="2">Wide Content</mat-grid-tile>
    </mat-grid-list>
  `,
})
export class BasicGridExample {
  // Component logic
}
```

### MatGridTile {Dependent Component}

**Purpose:** Individual grid tile that contains content and specifies its size requirements within the grid, supporting variable spans and automatic positioning.

**Interface:**

```typescript
interface MatGridTileProps {
  children: React.ReactNode; // Tile content and optional header/footer
  rowspan?: number; // Number of rows the tile spans (default: 1)
  colspan?: number; // Number of columns the tile spans (default: 1)
}

class MatGridTile {
  // Current position in grid (set by layout algorithm)
  _setStyle(property: string, value: any): void;

  // Size properties
  rowspan: number; // Rows occupied by this tile
  colspan: number; // Columns occupied by this tile
}
```

**Context Dependencies:** Grid layout context from parent MatGridList for positioning and sizing
**Parent Requirements:** Must be direct child of MatGridList component

**Usage Example:**

```typescript
<mat-grid-list cols="3" rowHeight="1:1">
  <!-- Single cell tile -->
  <mat-grid-tile>
    <img src="image1.jpg" alt="Image 1">
  </mat-grid-tile>

  <!-- Tile spanning 2 columns -->
  <mat-grid-tile [colspan]="2">
    <div class="large-content">
      <h3>Featured Content</h3>
      <p>This tile takes up two columns</p>
    </div>
  </mat-grid-tile>

  <!-- Tile spanning 2 rows -->
  <mat-grid-tile [rowspan]="2">
    <div class="tall-content">
      <h3>Tall Content</h3>
      <p>This tile spans two rows</p>
    </div>
  </mat-grid-tile>

  <!-- Large tile spanning multiple rows and columns -->
  <mat-grid-tile [colspan]="2" [rowspan]="2">
    <div class="hero-content">
      <h2>Hero Section</h2>
      <p>Large featured content area</p>
    </div>
  </mat-grid-tile>
</mat-grid-list>
```

## Component Relationships

### Data Flow

1. **MatGridList** receives configuration (cols, gutterSize, rowHeight) and calculates layout parameters
2. **TileCoordinator** algorithm processes all MatGridTile children to determine optimal positioning
3. **Positioning System** assigns specific (row, col) coordinates to each tile based on its size requirements
4. **Style Application** applies calculated positions and dimensions to tile elements in the DOM
5. **Responsive Updates** recalculate layout when configuration changes or container resizes

### Context Sharing

- **Grid Configuration**: MatGridList provides layout context (columns, gutters, row sizing) to all tiles
- **Positioning Algorithm**: Shared tile coordinator manages placement to avoid overlaps and optimize space usage
- **Style Coordination**: Grid container manages overall styling while tiles handle their individual content styles

### Event Handling

- **Layout Recalculation**: Automatic recalculation when tiles are added, removed, or resized
- **Responsive Changes**: Grid responds to container size changes and configuration updates
- **Tile Interactions**: Individual tiles can handle user interactions (clicks, hover) independently

## Best Practices

### Usage Guidelines

- **Define appropriate column count**: Choose column numbers that work well across different screen sizes (typically 2-6 for responsive design)
- **Use consistent tile sizes**: Maintain visual harmony by limiting the variety of tile sizes and following a clear sizing strategy
- **Optimize for content type**: Match row height configuration to your content - use aspect ratios for images, fixed heights for text content
- **Plan responsive behavior**: Consider how your grid will adapt to different screen sizes and adjust column counts accordingly

### Accessibility

- **Semantic content structure**: Ensure tile content uses proper heading hierarchy and semantic HTML elements
- **Keyboard navigation**: Provide keyboard navigation for interactive elements within tiles
- **Screen reader support**: Use proper alt text for images and descriptive content for complex tiles
- **Focus management**: Implement logical tab order for interactive tiles and maintain focus visibility

### Performance

- **Optimize large grids**: Use virtual scrolling or pagination for grids with many tiles to maintain performance
- **Lazy load images**: Implement lazy loading for image-heavy grids to improve initial load times
- **TrackBy functions**: Use trackBy functions with \*ngFor to optimize Angular's change detection for dynamic tiles
- **Efficient positioning**: The built-in positioning algorithm is optimized, but avoid frequent configuration changes during animations

    },
    // More items...
  ];

  getItemColspan(item: ContentItem): number {
    switch (item.type) {
      case "featured":
        return 4; // Takes most of the width
      case "article":
        return 2; // Medium size
      case "stat":
        return 1; // Small squares
      case "cta":
        return 3; // Medium-large
      default:
        return 1;
    }
  }

  getItemRowspan(item: ContentItem): number {
    switch (item.type) {
      case "featured":
        return 3; // Tall featured content
      case "article":
        return 2; // Medium height
      case "stat":
        return 1; // Single row
      case "cta":
        return 2; // Medium height
      default:
        return 1;
    }
  }
}

export class ProductShowcaseComponent implements OnInit {
  products: Product[] = [];
  responsiveCols = 4;
  responsiveGutter = '16px';

  @HostListener('window:resize', ['$event'])
  onResize(event: any) {
    this.updateResponsiveSettings();
  }

  ngOnInit() {
    this.updateResponsiveSettings();
  }

  private updateResponsiveSettings() {
    const width = window.innerWidth;
    if (width < 600) {
      this.responsiveCols = 2;
      this.responsiveGutter = '8px';
    } else if (width < 960) {
      this.responsiveCols = 3;
      this.responsiveGutter = '12px';
    } else if (width < 1280) {
      this.responsiveCols = 4;
      this.responsiveGutter = '16px';
    } else {
      this.responsiveCols = 5;
      this.responsiveGutter = '20px';
    }
  }

  getProductColspan(product: Product): number {
    // Featured products take more space
    return product.featured ? 2 : 1;
  }

  getStarArray(rating: number): number[] {
    return Array(Math.floor(rating)).fill(0);
  }
}
