# NavList Components

## Overview

### Purpose

Use this component to generate navigation list functionality

### When to Use

NavList components are ideal for creating navigation interfaces where users can navigate between different sections, pages, or routes within an application. Use them when you need to:

- **Primary navigation menus** - Create main navigation lists for application sections like dashboard, settings, profile, or feature areas
- **Sidebar navigation** - Build collapsible or persistent navigation panels with structured menu items and router integration
- **Breadcrumb-like navigation** - Implement hierarchical navigation lists that show the current path or allow navigation to parent sections
- **Category navigation** - Create organized navigation for content categories, filters, or organizational structures
- **Router-integrated menus** - Build navigation lists that integrate seamlessly with Angular Router for programmatic and declarative routing
- **Multi-level navigation** - Implement nested navigation structures with clear visual hierarchy and proper accessibility
- **Mobile navigation** - Create responsive navigation lists that work well in mobile drawer or bottom navigation patterns
- **Application shell navigation** - Build the core navigation framework for single-page applications with route state management

### Architecture

**No Coupling**: The NavList component is a standalone component that operates independently without requiring other components or shared context. It provides specialized navigation functionality with built-in Angular Router integration and can be used in isolation or combined with other layout components as needed.

### Component Interdependencies

The NavList component group has no interdependencies - MatNavList is a self-contained component that:

- Requires no parent components or containers
- Does not depend on shared context or state management
- Can be used independently or as a building block for navigation interfaces
- Provides specialized navigation capabilities with router integration
- Integrates seamlessly with Angular Router for declarative and programmatic navigation
- Supports accessibility features specifically designed for navigation patterns

## Components

### MatNavList {Standalone Component}

A specialized navigation list component that provides built-in Angular Router integration, active state management, and accessibility features optimized for navigation use cases.

**Purpose:** Provides the foundation for creating navigation interfaces where each item represents a route or navigation target, with automatic active state management, proper ARIA navigation attributes, and seamless router integration.

**Interface:**

```typescript
interface MatNavListProps {
  children: React.ReactNode; // Navigation content, typically mat-list-item elements with routerLink
  disableRipple?: boolean; // Disables ripple effects on navigation items (default: false)
  disabled?: boolean; // Disables the entire navigation list and all item interactions (default: false)
  id?: string; // Unique identifier for the navigation list
  class?: string; // CSS classes for additional styling
  role?: string; // ARIA role for accessibility (default: 'navigation')
}
```

**Usage Example:**

```tsx
import { MatNavList } from "@angular/material/list";

@Component({
  template: `
    <mat-nav-list>
      <mat-list-item routerLink="/dashboard" routerLinkActive="active-nav-item">
        <mat-icon matListItemIcon>dashboard</mat-icon>
        <span matListItemTitle>Dashboard</span>
        <span matListItemLine>View application overview</span>
      </mat-list-item>
      <mat-list-item routerLink="/projects" routerLinkActive="active-nav-item">
        <mat-icon matListItemIcon>folder</mat-icon>
        <span matListItemTitle>Projects</span>
        <span matListItemLine>Manage your projects</span>
      </mat-list-item>
      <mat-list-item routerLink="/settings" routerLinkActive="active-nav-item">
        <mat-icon matListItemIcon>settings</mat-icon>
        <span matListItemTitle>Settings</span>
        <span matListItemLine>Configure preferences</span>
      </mat-list-item>

export class BasicNavListExample {}

        *ngFor="let globalItem of globalNavigation"
        [routerLink]="globalItem.route"
        routerLinkActive="active-global-nav"
      >
        <mat-icon matListItemIcon>{{ globalItem.icon }}</mat-icon>
        <span matListItemTitle>{{ globalItem.title }}</span>
      </mat-list-item>
    </mat-nav-list>
  `,
  styles: [
    `
      .active-context-nav {
        background-color: rgba(var(--mat-accent-rgb), 0.12);
        border-left: 3px solid var(--mat-accent-500);
      }
      .active-global-nav {
        background-color: rgba(var(--mat-primary-rgb), 0.12);
        border-left: 3px solid var(--mat-primary-500);
      }
    `,
  ],
})
export class ContextualNavigationExample implements OnInit {
  currentContext: any = null;
  globalNavigation = [
    { route: "/dashboard", icon: "dashboard", title: "Dashboard" },
    { route: "/settings", icon: "settings", title: "Settings" },
  ];

  private navigationContexts = {
    "/projects": {
      title: "Project Tools",
      items: [
        { route: "/projects/board", icon: "view_kanban", title: "Board" },
        { route: "/projects/timeline", icon: "timeline", title: "Timeline" },
        { route: "/projects/files", icon: "folder", title: "Files" },
      ],
    },
    "/team": {
      title: "Team Management",
      items: [
        { route: "/team/members", icon: "people", title: "Members" },
        {
          route: "/team/permissions",
          icon: "admin_panel_settings",
          title: "Permissions",
        },
        { route: "/team/activity", icon: "history", title: "Activity" },
      ],
    },
  };

  constructor(private router: Router) {}

  ngOnInit(): void {
    this.router.events
      .pipe(filter((event) => event instanceof NavigationEnd))
      .subscribe((event: NavigationEnd) => {
        this.updateContextualNavigation(event.url);
      });

    // Set initial context
    this.updateContextualNavigation(this.router.url);
  }

  private updateContextualNavigation(url: string): void {
    const contextKey = Object.keys(this.navigationContexts).find((key) =>
      url.startsWith(key),
    );

    this.currentContext = contextKey
      ? this.navigationContexts[contextKey]
      : null;
  }
}
```

## Best Practices

### Usage Guidelines

- **Use for navigation only** - MatNavList is specifically designed for navigation patterns with router integration, unlike MatActionList which is for general actions
- **Implement proper routing** - Always use routerLink directives with navigation items to ensure proper browser history and accessibility
- **Leverage active state management** - Use routerLinkActive to provide visual feedback about the current page or section
- **Consider mobile responsiveness** - Design navigation patterns that work well on both desktop and mobile devices, potentially using drawer containers

### Accessibility

- **Navigation semantics** - MatNavList automatically provides proper ARIA navigation role and supports screen reader navigation patterns
- **Keyboard navigation** - Built-in support for arrow key navigation, Enter activation, and proper focus management for navigation flows
- **Route announcements** - Active route changes are automatically announced to screen readers when using proper router integration
- **Landmark navigation** - Use as primary navigation landmark to help users orient themselves within the application structure

### Performance

- **Route optimization** - Use exact route matching appropriately to prevent unnecessary active state calculations
- **TrackBy functions** - When rendering dynamic navigation items with \*ngFor, provide trackBy functions to optimize change detection
- **Lazy loading integration** - Works seamlessly with Angular's lazy loading modules for performance optimization
- **Memory management** - Clean up route subscriptions and observers properly to prevent memory leaks in navigation components
