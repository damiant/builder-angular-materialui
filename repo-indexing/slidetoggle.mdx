# SlideToggle Components

## Overview

### Purpose

Use this component to generate slide toggle functionality. The SlideToggle component provides Material Design styled toggle switches that enable users to switch between two states, typically representing "on" and "off" or "enabled" and "disabled". It follows Material Design 3 specifications for selection controls, offering intuitive visual feedback, proper accessibility features, and smooth animation transitions.

### When to Use

- **Feature Toggles**: When users need to enable or disable specific application features or settings (e.g., "Dark mode", "Enable notifications", "Auto-save")
- **Binary State Controls**: For representing and controlling boolean states with clear visual feedback (e.g., "Public/Private", "Active/Inactive", "Online/Offline")
- **Settings and Preferences**: In configuration panels where users need to toggle various options and see immediate visual confirmation of their choices
- **Privacy Controls**: For managing privacy settings where the state needs to be clearly visible (e.g., "Location sharing", "Data collection", "Profile visibility")
- **Accessibility Options**: When providing users with accessibility-related toggles (e.g., "High contrast", "Reduce motion", "Screen reader mode")
- **Form Controls**: In forms where binary choices need more visual prominence than checkboxes provide
- **Dashboard Controls**: For quick access controls in dashboards and admin panels (e.g., "Service status", "Maintenance mode", "API access")
- **Mobile-First Interfaces**: When designing for touch interfaces where toggles provide better usability than traditional checkboxes
- **Status Indicators**: For controls that represent and change system or item status with immediate visual feedback
- **Experimental Features**: When allowing users to opt-in or opt-out of beta features or experimental functionality

### Architecture

**No Coupling**: The SlideToggle component is a standalone component that operates independently without requiring other components. It provides:

- **Self-contained functionality**: Complete toggle implementation with all necessary features built-in including animation and state management
- **Flexible integration**: Can be used anywhere in the application without dependencies on parent or child components
- **Consistent behavior**: Maintains the same API and functionality regardless of context
- **Theme integration**: Automatically inherits from the Material Design theme system
- **Form integration**: Works seamlessly with Angular Reactive Forms and Template-driven Forms

## Components

### MatSlideToggle {Standalone Component}

A versatile slide toggle component that provides Material Design styling with smooth animations, comprehensive accessibility features, and flexible customization options for binary state controls.

**Purpose:** Creates interactive toggle switches with Material Design styling, smooth slide animations, proper accessibility features, and form integration for various binary choice scenarios.

**Interface:**

```typescript
interface MatSlideToggleProps {
  // State Management
  checked?: boolean; // Whether the slide toggle is in the "checked" (on) state

  // Interaction States
  disabled?: boolean; // Whether the slide toggle is disabled
  required?: boolean; // Marks the slide toggle as required in forms

  // Appearance and Styling
  color?: "primary" | "accent" | "warn"; // Theme color palette for the toggle
  labelPosition?: "before" | "after"; // Position of label relative to toggle switch
  disableRipple?: boolean; // Disables the ripple effect when toggle is interacted with

  // Accessibility
  ariaLabel?: string | null; // Accessible label for the toggle
  ariaLabelledby?: string | null; // ID of element that labels the toggle
  ariaDescribedby?: string | null; // ID of element that describes the toggle

  // Form Integration
  id?: string; // ID for the toggle element (auto-generated if not provided)
  name?: string | null; // Name attribute for form submission
  tabIndex?: number; // Tab order for keyboard navigation

  // Events
  change?: (event: MatSlideToggleChange) => void; // Fired when toggle state changes
  toggleChange?: () => void; // Fired when toggle is activated, regardless of state change
}

interface MatSlideToggleChange {
  source: MatSlideToggle; // Reference to the slide toggle instance
  checked: boolean; // New checked state of the toggle
}
```

**Methods:**

```typescript
interface MatSlideToggleMethods {
  focus(options?: FocusOptions): void; // Programmatically focus the toggle
  toggle(): void; // Programmatically toggle the checked state
}
```

**Usage Example:**

```typescript
import { MatSlideToggleModule } from "@angular/material/slide-toggle";
import { FormsModule, ReactiveFormsModule } from "@angular/forms";

@Component({
  selector: "app-slide-toggle-examples",
  template: `
    <!-- Basic slide toggle -->
    <mat-slide-toggle>Basic Toggle</mat-slide-toggle>

    <!-- Checked slide toggle -->
    <mat-slide-toggle [checked]="true">Pre-enabled Option</mat-slide-toggle>

    <!-- Disabled slide toggle -->
    <mat-slide-toggle [disabled]="true">Disabled Option</mat-slide-toggle>

    <!-- Slide toggle with colors -->
    <mat-slide-toggle color="primary" [checked]="true"
      >Primary Color</mat-slide-toggle
    >
    <mat-slide-toggle color="accent" [checked]="true"
      >Accent Color</mat-slide-toggle
    >
    <mat-slide-toggle color="warn" [checked]="true"
      >Warning Color</mat-slide-toggle
    >

    <!-- Slide toggle with label position -->
    <mat-slide-toggle labelPosition="before"
      >Label Before Toggle</mat-slide-toggle
    >

    <!-- Slide toggle with event handling -->
    <mat-slide-toggle (change)="onToggleChange($event)">
      Toggle with Event Handler
    </mat-slide-toggle>

    <!-- Two-way data binding -->
    <mat-slide-toggle [(ngModel)]="isEnabled">
      Feature Enabled: {{ isEnabled ? "On" : "Off" }}
    </mat-slide-toggle>

    <!-- Form integration -->
    <form [formGroup]="settingsForm">
      <mat-slide-toggle formControlName="enableNotifications" required>
        Enable Notifications *
      </mat-slide-toggle>
    </form>

    <!-- Accessibility enhanced -->
    <mat-slide-toggle
      [id]="'dark-mode-toggle'"
      [ariaLabel]="'Enable dark mode'"
      (change)="toggleDarkMode($event)"
    >
      Dark Mode
    </mat-slide-toggle>

    <!-- Slide toggle without ripple -->
    <mat-slide-toggle [disableRipple]="true">
      No Ripple Effect
    </mat-slide-toggle>
  `,
  imports: [MatSlideToggleModule, FormsModule, ReactiveFormsModule],
})
export class SlideToggleExamplesComponent {
  isEnabled = false;

  settingsForm = this.fb.group({
    enableNotifications: [false, Validators.required],
  });

  constructor(private fb: FormBuilder) {}

  onToggleChange(event: MatSlideToggleChange): void {
    console.log("Toggle changed:", event.checked);
  }

  toggleDarkMode(event: MatSlideToggleChange): void {
    console.log("Dark mode:", event.checked ? "enabled" : "disabled");
  }
}
```

    // Initialize with some toggles
    this.addToggle("Feature Alpha");
    this.addToggle("Feature Beta");
    this.addToggle("Feature Gamma");
  }

  get allEnabled(): boolean {
    const activeToggles = this.dynamicToggles.filter(
      (t) => !t.markedForRemoval,
    );
    return activeToggles.length > 0 && activeToggles.every((t) => t.enabled);
  }

  get someEnabled(): boolean {
    const activeToggles = this.dynamicToggles.filter(
      (t) => !t.markedForRemoval,
    );
    return activeToggles.some((t) => t.enabled) && !this.allEnabled;
  }

  addToggle(name?: string): void {
    const toggleName = name || `Feature ${this.nextId}`;
    this.dynamicToggles.push({
      id: this.nextId++,
      name: toggleName,
      enabled: false,
      markedForRemoval: false,
    });
  }

  markForRemoval(toggle: any): void {
    toggle.markedForRemoval = true;
    toggle.enabled = false;
  }

  removeSelected(): void {
    this.dynamicToggles = this.dynamicToggles.filter(
      (t) => !t.markedForRemoval,
    );
  }

  toggleAll(): void {
    const newState = !this.allEnabled;
    this.dynamicToggles
      .filter((t) => !t.markedForRemoval)
      .forEach((t) => (t.enabled = newState));

    console.log(`Toggle "${toggle.name}" (ID: ${toggle.id}):`, event.checked);
  }

  onBulkToggle(event: MatSlideToggleChange): void {
    this.dynamicToggles
      .filter((t) => !t.markedForRemoval)
      .forEach((t) => (t.enabled = event.checked));
  }

  getEnabledCount(): number {
    return this.dynamicToggles.filter((t) => !t.markedForRemoval && t.enabled)
      .length;
  }

  getDisabledCount(): number {
    return this.dynamicToggles.filter((t) => !t.markedForRemoval && !t.enabled)
      .length;
  }

  trackByFn(index: number, item: any): number {
    return item.id;

      this.conditionalForm.get("coreFeatureB")?.value
    );
  }

  private setupConditionalLogic(): void {
    // Master switch controls all features
    this.conditionalForm
      .get("masterSwitch")
      ?.valueChanges.subscribe((enabled) => {
        if (!enabled) {
          this.conditionalForm.patchValue({
            coreFeatureA: false,
            coreFeatureB: false,
          });
        }
      });

    // Core Feature A controls its extensions
    this.conditionalForm
      .get("coreFeatureA")
      ?.valueChanges.subscribe((enabled) => {
        if (!enabled) {
          this.conditionalForm.patchValue({
            extensionA1: false,
            extensionA2: false,
          });
        }
      });

    // All core features control experimental features
    this.conditionalForm.valueChanges.subscribe(() => {
      if (!this.allCoreEnabled) {
        this.conditionalForm.patchValue(
          {
            experimentalFeature1: false,
            experimentalFeature2: false,
          },
          { emitEvent: false },
        );
      }
    });
  }

  resetToSafeMode(): void {
    this.conditionalForm.reset({
      masterSwitch: true,
      coreFeatureA: true,
      coreFeatureB: false,
      extensionA1: false,
      extensionA2: false,
      experimentalFeature1: false,
      experimentalFeature2: false,
    });
  }

  enableAllRecommended(): void {
    this.conditionalForm.patchValue({
      masterSwitch: true,
      coreFeatureA: true,
      coreFeatureB: true,
      extensionA1: true,
      extensionA2: false,
      experimentalFeature1: false,
      experimentalFeature2: false,
    });
  }

  saveConfiguration(): void {
    if (this.conditionalForm.valid) {
      console.log("Saving configuration:", this.conditionalForm.value);

      // Show confirmation
      const enabledFeatures = Object.entries(this.conditionalForm.value)
        .filter(([key, value]) => value)
        .map(([key]) => key);

      console.log("Enabled features:", enabledFeatures);
    }
  }

## Best Practices

### Usage Guidelines

- **Use for binary states only**: Slide toggles are specifically designed for on/off or enabled/disabled states - use radio buttons or selects for multiple options
- **Provide immediate visual feedback**: Slide toggles should show their state clearly and respond to user interaction with smooth animations
- **Use descriptive labels**: Clearly indicate what state the toggle controls and what happens when it's activated
- **Consider toggle dependencies**: When toggles affect other toggles, provide clear visual indication of the relationship and disable dependent options appropriately
- **Maintain consistent behavior**: Ensure toggles behave predictably across your application with consistent positioning and styling

### Accessibility

- **Keyboard navigation**: Slide toggles automatically support keyboard navigation with Space key activation and Tab/Shift+Tab for focus movement
- **Screen reader support**: Use descriptive labels and aria-label attributes for toggles that need additional context
- **Focus management**: Ensure toggles receive proper focus indicators and maintain logical tab order
- **State communication**: Toggle states are automatically communicated to assistive technologies through proper ARIA attributes
- **Color independence**: Don't rely solely on color to indicate toggle state - the visual animation and position provide clear state indication

### Performance

- **Efficient change detection**: Use OnPush change detection strategy for components with many toggles to optimize performance
- **Debounced updates**: For toggles that trigger expensive operations, consider debouncing the change events to avoid excessive API calls
- **Conditional rendering**: Use \*ngIf to conditionally render dependent toggles and their related content to improve performance
- **TrackBy functions**: When rendering dynamic toggle lists, use trackBy functions to optimize change detection
- **Animation optimization**: Consider disabling ripple effects for performance-critical applications with many toggles
