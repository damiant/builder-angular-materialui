# ListItem Components

## Overview

### Purpose

Use this component group to generate list item components with optional sub-components

### When to Use

The ListItem component group is essential for creating structured, accessible list content across your application. Use this group when you need to:

- **Display structured data** - Present information with clear hierarchy using titles, descriptive lines, and metadata
- **Create interactive lists** - Build clickable list items for navigation, settings, or actions
- **Show content with media** - Display list items with icons, avatars, or images alongside text content
- **Implement user interfaces** - Create settings panels, contact lists, menu systems, or file browsers
- **Build responsive layouts** - Design lists that adapt to different screen sizes while maintaining readability
- **Ensure accessibility** - Leverage Angular Material's built-in accessibility features for screen readers and keyboard navigation

### Architecture

The ListItem component group follows a **moderate coupling** pattern where the primary `MatListItem` component serves as a flexible container that can work independently but is significantly enhanced by optional sub-components.

**Moderate Coupling Structure:**

- `MatListItem` functions as the foundational container that provides structure and styling
- Sub-components like `MatListItemTitle`, `MatListItemLine`, and `MatListItemMeta` enhance the primary component with specific content areas
- Each sub-component adds semantic meaning and specific styling to different parts of the list item
- Components can be mixed and matched based on content requirements

### Component Interdependencies

The ListItem group demonstrates a flexible composition pattern:

- **Primary Container**: `MatListItem` provides the foundational structure and handles accessibility
- **Content Enhancement**: Sub-components enhance specific areas within the container
- **Optional Composition**: Any combination of sub-components can be used based on content needs
- **Semantic Structure**: Each sub-component adds semantic meaning for better accessibility and styling

## Components

### MatListItem {Primary Component}

**Purpose:** Serves as the primary container for list item content, providing structure, styling, and accessibility features for list-based interfaces.

**Interface:**

```typescript
interface MatListItemProps {
  activated?: boolean; // Marks the list item as activated/selected
  disabled?: boolean; // Disables interaction with the list item
  lines?: number; // Controls the height based on number of text lines (1, 2, or 3)
  routerLink?: string | any[]; // Router navigation link
  routerLinkActive?: string; // CSS class for active router links
  click?: () => void; // Click event handler for interactive list items
  children?: React.ReactNode; // Content including sub-components
}
```

**Dependencies:** None - works independently as a standalone component
**Provides:** Container structure and accessibility foundation for list item content

**Usage Example:**

```tsx
import { MatListItem } from '@angular/material/list';

// Basic standalone usage
<mat-list-item>
  Simple list item content
</mat-list-item>

// With router navigation
<mat-list-item routerLink="/dashboard" routerLinkActive="active-nav-item">
  Dashboard
</mat-list-item>

// With click handler
<mat-list-item (click)="handleItemClick()">
  Interactive list item
</mat-list-item>
```

### MatListItemTitle {Content Enhancement Component}

**Purpose:** Designates the main title or primary text content of a list item, providing semantic structure and consistent typography.

**Interface:**

```typescript
interface MatListItemTitleProps {
  children: string | React.ReactNode; // The title text content
}
```

**Context Dependencies:** Works best within MatListItem container for proper styling and accessibility
**Parent Requirements:** Should be used within MatListItem for optimal semantic structure

**Usage Example:**

```tsx
// Essential for multi-line list items
<mat-list-item>
  <span matListItemTitle>Project Management Tool</span>
  <span matListItemLine>Organize and track your projects efficiently</span>
</mat-list-item>
```

### MatListItemLine {Content Enhancement Component}

**Purpose:** Provides secondary or tertiary text lines within a list item for additional descriptive information.

**Interface:**

```typescript
interface MatListItemLineProps {
  children: string | React.ReactNode; // The line text content
}
```

**Context Dependencies:** Enhances MatListItem with additional text lines
**Parent Requirements:** Must be used within MatListItem; maximum of two lines per list item

**Usage Example:**

```tsx
<mat-list-item>
  <span matListItemTitle>Document.pdf</span>
  <span matListItemLine>Important project documentation</span>
  <span matListItemLine>Modified: Today at 2:30 PM</span>
</mat-list-item>
```

### MatListItemMeta {Content Enhancement Component}

**Purpose:** Provides a designated area at the end of a list item for metadata, controls, badges, or additional interactive elements.

**Interface:**

```typescript
interface MatListItemMetaProps {
  children: React.ReactNode; // Meta content such as buttons, badges, or text
}
```

**Context Dependencies:** Positioned in the meta area of MatListItem
**Parent Requirements:** Should be used within MatListItem for proper positioning

**Usage Example:**

```tsx
<mat-list-item>
  <span matListItemTitle>Notifications</span>
  <span matListItemLine>Receive app notifications</span>
  <mat-slide-toggle matListItemMeta [checked]="notificationsEnabled"></mat-slide-toggle>
</mat-list-item>
```

### MatListItemAvatar {Visual Enhancement Component}

**Purpose:** Displays avatar images or circular visual elements at the beginning of a list item.

**Interface:**

```typescript
interface MatListItemAvatarProps {
  src?: string; // Image source URL
  alt?: string; // Alternative text for accessibility
  children?: React.ReactNode; // Custom avatar content
}
```

**Context Dependencies:** Positioned in the leading area of MatListItem
**Parent Requirements:** Should be used within MatListItem for proper positioning and styling

**Usage Example:**

```tsx
<mat-list-item>
  <img matListItemAvatar [src]="user.avatar" [alt]="user.name">
  <span matListItemTitle>{{user.name}}</span>
  <span matListItemLine>{{user.email}}</span>
</mat-list-item>
```

### MatListItemIcon {Visual Enhancement Component}

**Purpose:** Displays icons at the beginning of a list item to provide visual context and improve content recognition.

**Interface:**

```typescript
interface MatListItemIconProps {
  color?: "primary" | "accent" | "warn"; // Material Design color theme
  children: React.ReactNode; // Icon content, typically mat-icon
}
```

**Context Dependencies:** Positioned in the leading area of MatListItem
**Parent Requirements:** Should be used within MatListItem for proper positioning

**Usage Example:**

```tsx
<mat-list-item>
  <mat-icon matListItemIcon>settings</mat-icon>
  <span matListItemTitle>Settings</span>
  <span matListItemLine>Configure application preferences</span>
</mat-list-item>
```

## Integration Patterns

### Basic Usage

```tsx
import { MatList, MatListItem } from '@angular/material/list';

// Minimal list item
<mat-list>
  <mat-list-item>
    <span matListItemTitle>Simple Item</span>
  </mat-list-item>
</mat-list>

// Multi-line list item
<mat-list>
  <mat-list-item>
    <span matListItemTitle>Multi-line Item</span>
    <span matListItemLine>Additional description text</span>
  </mat-list-item>
</mat-list>
```

### Advanced Usage

```tsx
// Complex list item with all features
<mat-list>
  <mat-list-item (click)="handleItemClick()" [lines]="3">
    <mat-icon matListItemIcon color="primary">folder</mat-icon>
    <span matListItemTitle>Project Folder</span>
    <span matListItemLine>Contains all project files and documentation</span>
    <span matListItemLine>Last modified: {{project.lastModified | date}}</span>
    <mat-icon matListItemMeta>chevron_right</mat-icon>
  </mat-list-item>
</mat-list>

// Navigation list item
<mat-list>
  <mat-list-item routerLink="/dashboard" routerLinkActive="active-nav-item">
    <mat-icon matListItemIcon>dashboard</mat-icon>
    <span matListItemTitle>Dashboard</span>
    <span matListItemLine>View application overview</span>
  </mat-list-item>
</mat-list>
```

## Component Relationships

### Data Flow

- **MatListItem** serves as the primary data container and event handler
- **Content components** (Title, Line) receive and display text data
- **Visual components** (Icon, Avatar) enhance the visual presentation
- **Meta components** provide additional context or interactive elements

### Context Sharing

- All sub-components inherit styling context from the parent MatListItem
- Accessibility attributes are coordinated between the container and sub-components
- Responsive behavior is managed at the MatListItem level

### Event Handling

- Click events are typically handled at the MatListItem level
- Meta components can have their own event handlers for specific interactions
- Event propagation can be controlled to prevent conflicts between item and meta actions

## Best Practices

### Usage Guidelines

- **Use semantic structure** - Always include `matListItemTitle` for multi-line items to ensure proper accessibility and visual hierarchy
- **Limit line content** - Restrict to maximum two `matListItemLine` elements per item to maintain readability and performance
- **Choose appropriate visual elements** - Use either `matListItemIcon` or `matListItemAvatar`, not both, to avoid visual clutter
- **Design for interaction** - When list items are clickable, provide clear visual feedback and appropriate cursor states
- **Organize meta content** - Use `matListItemMeta` for supplementary information or controls that don't interfere with primary content

### Accessibility

- **Provide descriptive titles** - Use `matListItemTitle` to give screen readers clear identification of each list item
- **Include alternative text** - Always provide `alt` attributes for avatar images to support screen readers
- **Maintain focus management** - Ensure keyboard navigation works smoothly between interactive list items
- **Use ARIA labels** - Leverage Angular Material's built-in ARIA support for complex list interactions

### Performance

- **Optimize for large lists** - Use Angular's `*ngFor` with `trackBy` functions for lists with many items
- **Lazy load images** - Implement lazy loading for avatar images in long lists to improve initial page load
- **Debounce interactions** - For frequently updated lists, debounce click handlers to prevent excessive event processing

## Common Patterns

### Interactive Action Lists

```tsx
// Settings panel with various control types
<mat-list>
  <mat-list-item (click)="toggleNotifications()">
    <mat-icon matListItemIcon>notifications</mat-icon>
    <span matListItemTitle>Push Notifications</span>
    <span matListItemLine>Receive app notifications</span>
    <mat-slide-toggle matListItemMeta [checked]="notificationsEnabled"></mat-slide-toggle>
  </mat-list-item>

  <mat-list-item (click)="changePassword()">
    <mat-icon matListItemIcon>lock</mat-icon>
    <span matListItemTitle>Change Password</span>
    <mat-icon matListItemMeta>chevron_right</mat-icon>
  </mat-list-item>
</mat-list>
```

### Navigation Lists with Status

```tsx
// Navigation with badge indicators
<mat-list>
  <mat-list-item routerLink="/messages" routerLinkActive="active-nav-item">
    <mat-icon matListItemIcon>mail</mat-icon>
    <span matListItemTitle>Messages</span>
    <span matListItemLine>View your messages</span>
    <mat-badge matListItemMeta [content]="unreadCount" *ngIf="unreadCount > 0"></mat-badge>
  </mat-list-item>
</mat-list>
```

### Dynamic Content Lists

```tsx
// File browser with conditional meta content
<mat-list>
  <mat-list-item *ngFor="let item of items" (click)="selectItem(item)">
    <mat-icon matListItemIcon [color]="getItemColor(item.type)">
      {{getItemIcon(item.type)}}
    </mat-icon>
    <span matListItemTitle>{{item.name}}</span>
    <span matListItemLine>{{item.description}}</span>
    <span matListItemLine>Last updated: {{item.lastModified | date}}</span>
    <mat-chip matListItemMeta [color]="getStatusColor(item.status)">
      {{item.status}}
    </mat-chip>
  </mat-list-item>
</mat-list>
```
