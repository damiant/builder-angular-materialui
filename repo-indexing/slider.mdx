# Slider Components

## Overview

### Purpose

Use this component to generate slider functionality. The Slider component provides Material Design styled range sliders that enable users to select numerical values from a continuous or discrete range by dragging a handle along a track. It follows Material Design 3 specifications for input controls, offering consistent visual styling, proper accessibility features, and seamless form integration.

### When to Use

- **Numerical Range Selection**: When users need to select a numerical value from a continuous range (e.g., volume control, price range, age selection)
- **Settings and Configuration**: For adjusting configuration values like brightness, quality settings, or performance parameters
- **Filter Controls**: When providing range-based filters for search results or data visualization
- **Media Controls**: For controlling audio/video playback position, volume, or playback speed
- **Data Input**: When collecting numerical data where visual feedback of the value's position within a range is helpful
- **Progressive Adjustment**: For settings that benefit from real-time feedback as the user adjusts the value
- **Mobile-Friendly Input**: When creating touch-friendly numerical input that works well on mobile devices
- **Accessible Value Selection**: When you need properly structured, keyboard-navigable, and screen-reader friendly numerical input controls
- **Visual Value Comparison**: When users need to understand the relative position of a value within its possible range

### Architecture

**No Coupling**: The Slider component is a standalone component that operates independently without requiring other components beyond optional form field integration. It provides:

- **Self-contained functionality**: Complete slider implementation with track, handle, value display, and interaction handling
- **Flexible value management**: Supports continuous and discrete values, custom step intervals, and value formatting
- **Form integration**: Designed to work seamlessly with reactive forms and template-driven forms
- **Theme integration**: Automatically inherits from the Material Design theme system

## Components

### MatSlider {Standalone Component}

A versatile slider component that provides Material Design styling with comprehensive value selection and accessibility support.

**Purpose:** Creates interactive range sliders with Material Design styling, proper accessibility features, and seamless form integration for selecting numerical values from defined ranges.

**Interface:**

```typescript
interface MatSliderProps {
  // Value Management
  value?: number; // Current slider value
  min?: number; // Minimum value (default: 0)
  max?: number; // Maximum value (default: 100)
  step?: number; // Step interval (default: 1)

  // Display Options
  thumbLabel?: boolean; // Show value label on thumb
  tickInterval?: number | "auto"; // Tick mark intervals
  displayWith?: (value: number) => string; // Custom value display function

  // Interaction States
  disabled?: boolean; // Whether the slider is disabled
  invert?: boolean; // Invert the slider direction
  vertical?: boolean; // Vertical orientation

  // Styling
  color?: "primary" | "accent" | "warn"; // Theme color
  disableRipple?: boolean; // Disable ripple effects

  // Form Integration
  tabIndex?: number; // Tab order for keyboard navigation
  id?: string; // Unique identifier
  ariaLabel?: string; // Accessibility label
  ariaLabelledby?: string; // ID of labeling element

  // Events
  change?: (event: MatSliderChange) => void; // Value change event (on release)
  input?: (event: MatSliderChange) => void; // Continuous input event (while dragging)
}

interface MatSliderChange {
  source: MatSlider; // The slider component instance
  value: number; // The new slider value
}
```

  onSensitivityInput(event: MatSliderChange): void {
    this.sensitivity = event.value;
    this.updateSensitivityFeedback(event.value);
  }

  onSensitivityChange(event: MatSliderChange): void {
    console.log("Final sensitivity value:", event.value);
  }

  private updateSensitivityFeedback(value: number): void {
    if (value < 0.5) {
      this.sensitivityFeedback = "Very low sensitivity";
    } else if (value < 1.0) {
      this.sensitivityFeedback = "Low sensitivity";
    } else if (value === 1.0) {
      this.sensitivityFeedback = "Normal sensitivity";
    } else if (value < 1.5) {
      this.sensitivityFeedback = "High sensitivity";
    } else {
      this.sensitivityFeedback = "Very high sensitivity";
    }
  }
}

      .slider-examples {

      .slider-section {
        margin-bottom: 32px;
        padding: 16px;
        background: #f9f9f9;
        border-radius: 8px;
      }

      .slider-section h3 {
        margin: 0 0 16px 0;
        color: #333;
        font-size: 18px;
      }

      .slider-section p {
        margin: 12px 0 0 0;
        color: #666;
        font-weight: 500;
      }

      .volume-slider,

      .budget-slider,

export class BasicSlidersComponent {
  volume = 75;
  rating = 4;
  budget = 800;
  progress = 65;

  formatCurrency(value: number): string {
    return new Intl.NumberFormat("en-US", {
      style: "currency",
      currency: "USD",
      minimumFractionDigits: 0,
    }).format(value);
  }

  formatPercentage(value: number): string {
    return `${value}%`;
  }

  getRatingText(rating: number): string {
    const ratings = {
      1: "⭐ Poor",
      2: "⭐⭐ Fair",
      3: "⭐⭐⭐ Good",
      4: "⭐⭐⭐⭐ Very Good",
      5: "⭐⭐⭐⭐⭐ Excellent",
    };
    return ratings[rating as keyof typeof ratings] || "Not rated";
  }
}

## Best Practices

### Usage Guidelines

- **Choose appropriate ranges**: Set min/max values that make sense for your use case and provide reasonable default values
- **Use meaningful step intervals**: Set step values that provide appropriate precision without making the slider too sensitive
- **Provide visual feedback**: Use thumb labels, tick marks, and value displays to help users understand the current value
- **Consider the use case**: Use continuous sliders for values like volume or brightness, discrete sliders for settings with specific levels
- **Include formatting functions**: Use `displayWith` to format values in a user-friendly way (e.g., percentages, currency, units)
- **Handle real-time updates**: Use the `input` event for real-time feedback and `change` event for final value confirmation

### Accessibility

- **ARIA labeling**: Sliders automatically provide proper ARIA roles and attributes for screen readers
- **Keyboard navigation**: Supports arrow keys for fine adjustment, Page Up/Down for larger steps, Home/End for min/max values
- **Screen reader support**: Properly announces current values, ranges, and value changes
- **Focus management**: Maintains proper focus indicators and supports keyboard-only navigation
- **High contrast support**: All slider components work correctly with high contrast themes

### Performance

- **Debounce input events**: For expensive operations, consider debouncing the `input` event to avoid excessive updates
- **Optimize rendering**: Use OnPush change detection strategy and trackBy functions for dynamic slider lists
- **Value formatting**: Implement efficient formatting functions that don't perform expensive calculations
- **Form integration**: Use reactive forms with proper validation strategies to optimize form performance
- **Bundle optimization**: Import only the slider module to minimize bundle size
