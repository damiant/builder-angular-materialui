# Icon Components

## Overview

### Purpose

Use this component to generate icon functionality

### When to Use

Icon components are essential visual elements that enhance user interface communication and provide intuitive visual cues throughout your application. Use them when you need to:

- **Visual communication** - Convey meaning and actions through universally recognized symbols and graphics
- **Space-efficient UI** - Replace text labels with compact visual representations in toolbars, buttons, and navigation
- **User guidance** - Provide visual cues for actions, status, and navigation to improve user experience
- **Brand consistency** - Display logos, brand icons, and custom graphics that align with your design system
- **Interactive elements** - Enhance buttons, links, and form controls with meaningful visual indicators
- **Status indication** - Show loading states, success/error states, and progress indicators
- **Navigation aids** - Create visual hierarchy and wayfinding through menu icons, arrows, and directional indicators
- **Content categorization** - Use icons to represent different content types, file formats, or data categories
- **Accessibility enhancement** - Provide visual support for users while maintaining screen reader compatibility
- **International applications** - Use universal symbols that transcend language barriers
- **Dashboard elements** - Create data visualizations and metric displays with appropriate iconography

### Architecture

**No Coupling**: The Icon component is a standalone component that can be used independently throughout your application. It provides flexible icon display capabilities from multiple sources including SVG icons, font-based icon sets, and custom graphics without requiring any parent or child components.

### Component Interdependencies

The Icon component demonstrates no coupling as it is a self-contained component that:

- **Independent usage**: Can be placed anywhere in your application without dependencies on other components
- **Flexible integration**: Works seamlessly with buttons, form fields, menus, and other Material Design components
- **Multiple icon sources**: Supports both SVG icons and font-based icon sets for maximum flexibility
- **Theme integration**: Automatically inherits theming and color palettes from Angular Material's theming system

## Components

### MatIcon {Standalone Component}

A versatile component for displaying icons from various sources including SVG icon sets, font-based icon libraries, and inline SVG graphics with full Material Design theming and accessibility support.

**Purpose:** Provides a consistent, accessible way to display icons throughout your application with support for multiple icon sources, theming integration, and responsive sizing while maintaining performance and accessibility standards.

**Interface:**

```typescript
interface MatIconProps {
  // Icon Source Configuration
  svgIcon?: string; // Name of SVG icon from registered icon sets (e.g., 'home', 'menu')
  fontIcon?: string; // Name of icon from font-based icon set (e.g., 'material-icons')
  fontSet?: string; // Font set to use for font-based icons (default: 'material-icons')

  // Styling and Appearance
  color?: ThemePalette; // Theme color palette ('primary', 'accent', 'warn')
  inline?: boolean; // Whether to display icon inline with text (default: false)

  // Accessibility
  ariaHidden?: boolean; // Hide from screen readers when decorative (default: true)
  ariaLabel?: string; // Accessible label for screen readers

  // Standard Attributes
  class?: string; // Additional CSS classes
  style?: React.CSSProperties; // Inline styles
  id?: string; // Unique identifier

  // Events
  onClick?: (event: MouseEvent) => void; // Click event handler
}

interface IconRegistry {
  // Icon registration methods for custom SVG icons
  addSvgIcon(iconName: string, url: SafeResourceUrl): IconRegistry;
  addSvgIconSet(url: SafeResourceUrl): IconRegistry;
  addSvgIconInNamespace(
    namespace: string,
    iconName: string,
    url: SafeResourceUrl,
  ): IconRegistry;
}
```

### Custom SVG Icon Integration

```tsx
import { MatIconRegistry } from "@angular/material/icon";
import { DomSanitizer } from "@angular/platform-browser";

// Register custom SVG icons
@Component({
  template: `
    <div class="custom-icons">
      <mat-icon svgIcon="custom-logo"></mat-icon>
      <mat-icon svgIcon="company:analytics-chart"></mat-icon>
      <mat-icon svgIcon="company:user-profile"></mat-icon>
    </div>
  `,
})
export class CustomIconExample {
  constructor(
    private iconRegistry: MatIconRegistry,
    private sanitizer: DomSanitizer,
  ) {
    // Register individual SVG icons
    this.iconRegistry.addSvgIcon(
      "custom-logo",
      this.sanitizer.bypassSecurityTrustResourceUrl("/assets/icons/logo.svg"),
    );

    // Register namespaced icon set
    this.iconRegistry.addSvgIconSetInNamespace(
      "company",
      this.sanitizer.bypassSecurityTrustResourceUrl(
        "/assets/icons/company-icons.svg",
      ),
    );
  }
}
```

## Best Practices

### Usage Guidelines

- **Choose meaningful icons**: Select icons that clearly represent their intended function or content to avoid user confusion
- **Maintain consistency**: Use the same icon for the same action throughout your application to build user familiarity
- **Consider cultural context**: Be aware that icon meanings can vary across cultures and regions
- **Provide alternative text**: Always include appropriate labels or tooltips for accessibility when icons are interactive
- **Use appropriate sizing**: Ensure icons are large enough to be easily recognizable but not overwhelming in the interface

### Accessibility

- **Screen reader support**: Use `aria-label` for interactive icons and `aria-hidden="true"` for purely decorative icons
- **Keyboard navigation**: Ensure interactive icons are accessible via keyboard navigation with proper focus management
- **Color contrast**: Maintain sufficient color contrast between icons and their background for users with visual impairments
- **Alternative text**: Provide text alternatives or labels for icons that convey important information
- **Focus indicators**: Ensure interactive icons have clear focus indicators for keyboard users

### Performance

- **Icon optimization**: Use SVG icons when possible for scalability and smaller file sizes compared to raster images
- **Icon fonts**: Consider using icon fonts for common icons to reduce HTTP requests and improve loading performance
- **Lazy loading**: For applications with many custom SVG icons, consider lazy loading icon sets to improve initial load time
- **Caching strategy**: Implement proper caching for custom icon assets to reduce repeated downloads
- **Bundle optimization**: Only include the icon sets you actually use to minimize bundle size

### Theming and Customization

- **Theme integration**: Use Angular Material's color palette properties (`primary`, `accent`, `warn`) for consistent theming
- **Custom colors**: Define custom CSS classes for specific icon colors that align with your brand guidelines
- **Size consistency**: Establish a consistent icon sizing scale and use CSS classes to maintain uniformity
- **Icon registration**: Register custom SVG icons once during application initialization for optimal performance
- **Responsive behavior**: Ensure icons scale appropriately across different device sizes and screen densities
