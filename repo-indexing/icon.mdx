# Icon Components

## Overview

### Purpose

Use this component to generate icon functionality. The Icon component provides a comprehensive solution for displaying icons in your Angular Material application, supporting both SVG and font-based icons with consistent styling, proper accessibility features, and flexible customization options that integrate seamlessly with Material Design themes.

### When to Use

- **UI Navigation Elements**: When creating navigation menus, toolbars, and action bars that require visual indicators alongside or instead of text labels
- **Button Enhancement**: For adding visual context to buttons, making actions more intuitive and recognizable to users
- **Status Indicators**: When displaying system states, notifications, or feedback messages that benefit from visual reinforcement
- **Content Organization**: For categorizing content sections, indicating file types, or providing visual hierarchy in lists and tables
- **Interactive Elements**: When creating icon-only buttons for space-constrained interfaces like mobile applications or compact toolbars
- **Data Visualization**: For representing different data types, categories, or actions in charts, graphs, and data tables
- **Form Controls**: When enhancing form inputs with contextual icons that clarify field purpose or indicate validation states
- **Accessibility Enhancement**: For providing visual cues that support screen readers and improve overall accessibility
- **Brand Consistency**: When maintaining visual consistency across the application with standardized iconography
- **Multi-language Support**: For creating interfaces that rely less on text and more on universally understood visual symbols

### Architecture

**No Coupling**: The Icon component is a standalone component that operates independently without requiring other components. It provides:

- **Self-contained functionality**: Complete icon rendering with built-in support for multiple icon sources and formats
- **Flexible integration**: Can be embedded anywhere in the application without dependencies on parent or child components
- **Consistent behavior**: Maintains the same API and visual consistency regardless of usage context
- **Theme integration**: Automatically inherits styling from the Material Design theme system
- **Accessibility features**: Built-in ARIA support and semantic HTML structure for screen reader compatibility

## Components

### MatIcon {Standalone Component}

A versatile icon component that provides Material Design icon rendering with support for font icons, SVG icons, and custom icon sets, offering comprehensive customization and accessibility features.

**Purpose:** Renders icons from various sources (font icons, SVG icons, custom icon sets) with Material Design styling, proper accessibility features, and seamless integration with Angular Material's theming system.

**Interface:**

```typescript
interface MatIconProps {
  // Icon Source Configuration
  fontIcon?: string; // Name of the icon within the specified font set
  fontSet?: string; // Font set that the icon is part of (e.g., 'material-icons')
  svgIcon?: string; // Name of the icon in the SVG icon set

  // Styling and Appearance
  color?: string | null; // Theme color palette (M2 themes only, deprecated in M3)
  inline?: boolean; // Whether to inline the icon, sizing to match surrounding font

  // Accessibility
  ariaLabel?: string; // Accessible label for screen readers
  ariaHidden?: boolean; // Whether to hide from screen readers

  // Standard HTML attributes (inherited from HTMLElement)
  class?: string; // CSS classes for custom styling
  style?: string; // Inline styles for specific customization
  id?: string; // Unique identifier
  tabIndex?: number; // Tab order for keyboard navigation
}

// Configuration interfaces for icon management
interface MatIconDefaultOptions {
  color?: string; // Default color for all icons
  fontSet?: string; // Default font set for all icons
}

interface IconOptions {
  viewBox?: string; // SVG viewBox attribute
  withCredentials?: boolean; // HTTP credentials for icon fetching
}

interface SafeResourceUrlWithIconOptions {
  url: SafeResourceUrl; // Sanitized URL for the icon resource
  options?: IconOptions; // Additional options for the icon
}
```

**Usage Example:**

```typescript
import { MatIconModule } from "@angular/material/icon";
import { MatIconRegistry } from "@angular/material/icon";
import { DomSanitizer } from "@angular/platform-browser";

@Component({
  selector: "app-icon-examples",
  template: `
    <!-- Basic Material Icons font icon -->
    <mat-icon>home</mat-icon>
    <mat-icon>search</mat-icon>
    <mat-icon>settings</mat-icon>

    <!-- Font icon with specific font set -->
    <mat-icon fontSet="material-icons-outlined" fontIcon="favorite"></mat-icon>

    <!-- Inline icon that matches surrounding text size -->
    <p>Click the <mat-icon inline>info</mat-icon> icon for more details</p>

    <!-- SVG icon from registered icon set -->
    <mat-icon svgIcon="custom-logo"></mat-icon>

    <!-- Icon with custom styling -->
    <mat-icon class="large-icon">star</mat-icon>

    <!-- Icon with accessibility label -->
    <mat-icon aria-label="Delete item">delete</mat-icon>

    <!-- Decorative icon hidden from screen readers -->
    <mat-icon aria-hidden="true">decorative</mat-icon>

    <!-- Icon with theme color (M2 themes only) -->
    <mat-icon color="primary">bookmark</mat-icon>
  `,
  styles: [
    `
      .large-icon {
        font-size: 48px;
        width: 48px;
        height: 48px;
      }
    `,
  ],
  imports: [MatIconModule],
})
export class IconExamplesComponent {
  constructor(
    private matIconRegistry: MatIconRegistry,
    private domSanitizer: DomSanitizer,
  ) {
    // Register custom SVG icon
    this.matIconRegistry.addSvgIcon(
      "custom-logo",
      this.domSanitizer.bypassSecurityTrustResourceUrl("assets/icons/logo.svg"),
    );

    // Register icon set
    this.matIconRegistry.addSvgIconSet(
      this.domSanitizer.bypassSecurityTrustResourceUrl(
        "assets/icons/icon-set.svg",
      ),
    );
  }
}
```

    );

    this.matIconRegistry.addSvgIconSetInNamespace(
      "custom",
      this.domSanitizer.bypassSecurityTrustResourceUrl(
        "assets/icons/custom-set.svg",
      ),
    );
  }

## Best Practices

### Usage Guidelines

- **Choose the right icon source**: Use Material Icons for consistency with Material Design, custom SVG icons for brand-specific elements, and ensure all icons follow the same visual style
- **Maintain appropriate sizing**: Keep icons at standard sizes (16px, 20px, 24px, 32px, 48px) and use the `inline` property when icons should match surrounding text size
- **Provide meaningful context**: Always include accessible labels for icon-only buttons and ensure icons clearly represent their intended actions or content
- **Optimize for touch targets**: Ensure interactive icons meet minimum touch target sizes (44px) and provide adequate spacing between clickable icons
- **Use consistent iconography**: Establish a consistent icon vocabulary across your application and avoid using different icons for the same action in different contexts

### Accessibility

- **Screen reader support**: Use `aria-label` attributes for interactive icons and `aria-hidden="true"` for purely decorative icons that don't convey meaningful information
- **Keyboard navigation**: Ensure icon buttons are properly focusable and provide visible focus indicators for keyboard users
- **Color independence**: Don't rely solely on color to convey meaning; combine color changes with icon shape changes or additional text labels
- **Context and meaning**: Provide tooltips or adjacent text labels to clarify icon meaning, especially for less universal icon symbols
- **High contrast**: Ensure icons maintain sufficient contrast ratios against their backgrounds for users with visual impairments

### Performance

- **Icon loading optimization**: Register frequently used SVG icons at application startup and consider lazy loading for less common icon sets
- **Bundle size management**: Import only the icon fonts you need and consider using tree-shaking with individual icon imports when possible
- **Caching strategies**: Implement proper caching headers for icon font files and SVG assets to improve loading performance on repeat visits
- **Memory efficiency**: Avoid creating excessive DOM nodes for icons by reusing icon components and implementing proper change detection strategies
- **Rendering optimization**: Use CSS transforms instead of JavaScript animations for icon transitions and leverage GPU acceleration for smooth animations
