# RadioGroup Components

## Overview

### Purpose

Use this component group to generate radio button group functionality with required hierarchical structure. The RadioGroup component group provides Material Design styled radio button groups that enable users to select exactly one option from a set of mutually exclusive choices. It follows Material Design 3 specifications for selection controls, offering consistent visual styling, proper accessibility features, and seamless form integration.

### When to Use

- **Single Selection from Multiple Options**: When users need to select exactly one option from a group of 2-7 mutually exclusive choices
- **Form Data Entry**: For collecting structured data in forms where only one option should be selected from predefined choices
- **Settings and Preferences**: When providing configuration options where only one setting can be active at a time
- **Survey and Questionnaire Forms**: For survey questions that require a single answer from multiple choices
- **Filter Controls**: When creating filter interfaces where only one filter option should be active
- **Toggle Between States**: For interfaces that need to switch between different modes or states
- **Payment or Shipping Options**: When users need to select one payment method or shipping option from available choices
- **Account Type Selection**: For registration forms where users choose between different account types or subscription plans
- **Accessibility-First Selection**: When you need properly structured, keyboard-navigable, and screen-reader friendly exclusive selection controls

### Architecture

**Critical/Strong Coupling**: The RadioGroup component group follows a required hierarchical structure where components must be used together in specific parent-child relationships. The architecture is built around:

- **Required parent-child relationships**: `mat-radio-group` serves as the mandatory container for all radio button components
- **Shared context dependencies**: The radio group manages mutual exclusivity, selection state, and form integration
- **Compositional hierarchy**: Individual radio buttons must be children of the radio group to function properly

### Component Interdependencies

The RadioGroup component group requires strict composition patterns:

- **`mat-radio-group`** → Required root container that manages selection state, form integration, and accessibility
- **`mat-radio-button`** → Individual radio button components that must be direct children of `mat-radio-group`
- **Selection Management** → Radio group ensures only one button can be selected at a time
- **Form Integration** → Radio group provides form control interface and validation support
- **Accessibility Context** → Radio group manages ARIA attributes and keyboard navigation for all child buttons

## Components

### MatRadioGroup {Primary Container Component}

The mandatory root container component that manages radio button selection, form integration, and accessibility for all child radio buttons.

**Purpose:** Serves as the foundational wrapper that coordinates selection state, ensures mutual exclusivity, provides form control interface, and manages accessibility attributes for all contained radio button components.

**Interface:**

```typescript
interface MatRadioGroupProps {
  // Selection Management
  value?: any; // Currently selected value
  name?: string; // Name attribute for form submission
  selected?: MatRadioButton; // Currently selected radio button reference

  // Form Integration
  required?: boolean; // Whether selection is required
  disabled?: boolean; // Whether the entire group is disabled

  // Layout and Styling
  labelPosition?: "before" | "after"; // Label position for all radio buttons
  color?: "primary" | "accent" | "warn"; // Theme color for all radio buttons

  // Events
  change?: (event: MatRadioChange) => void; // Selection change event
}

interface MatRadioChange {
  source: MatRadioButton; // The selected radio button
  value: any; // The value of the selected radio button
}
```

**Dependencies:** Requires Angular Material's radio module and at least one child `mat-radio-button`
**Provides:** Selection management, form control interface, and accessibility context for all child radio buttons

**Usage Example:**

```typescript
import { MatRadioModule } from "@angular/material/radio";
import { ReactiveFormsModule, FormControl } from "@angular/forms";

@Component({
  selector: "app-radio-group-example",
  template: `
    <mat-radio-group
      [(value)]="selectedOption"
      (change)="onSelectionChange($event)"
    >
      <mat-radio-button value="option1">Option 1</mat-radio-button>
      <mat-radio-button value="option2">Option 2</mat-radio-button>
      <mat-radio-button value="option3">Option 3</mat-radio-button>
    </mat-radio-group>

    <!-- With form control -->
    <mat-radio-group [formControl]="paymentMethod">
      <mat-radio-button value="credit">Credit Card</mat-radio-button>
      <mat-radio-button value="debit">Debit Card</mat-radio-button>
      <mat-radio-button value="paypal">PayPal</mat-radio-button>
    </mat-radio-group>
  `,
  imports: [MatRadioModule, ReactiveFormsModule],
})
export class RadioGroupExampleComponent {
  selectedOption = "option2";
  paymentMethod = new FormControl("credit");

  onSelectionChange(event: MatRadioChange): void {
    console.log("Selected:", event.value);
  }
}
```

### MatRadioButton {Individual Radio Button Component}

Individual radio button components that represent selectable options within the radio group context.

**Purpose:** Provides individual selectable options with Material Design styling, accessibility support, and integration with the parent radio group's selection management.

**Interface:**

```typescript
interface MatRadioButtonProps {
  // Identification and Value
  value?: any; // The value associated with this radio button
  id?: string; // Unique identifier
  name?: string; // Name attribute (inherited from radio group)

  // State Management
  checked?: boolean; // Whether this radio button is selected
  disabled?: boolean; // Whether this radio button is disabled
  required?: boolean; // Whether selection is required (inherited from radio group)

  // Styling and Layout
  labelPosition?: "before" | "after"; // Position of the label relative to the radio circle
  color?: "primary" | "accent" | "warn"; // Theme color
  disableRipple?: boolean; // Whether to disable ripple effects

  // Accessibility
  ariaLabel?: string; // Accessibility label
  ariaLabelledby?: string; // ID of element that labels this radio button
  ariaDescribedby?: string; // ID of element that describes this radio button

  // Events
  change?: (event: MatRadioChange) => void; // Selection change event
}
```

**Context Dependencies:** Must be used within `mat-radio-group`
**Parent Requirements:** Required parent component is `mat-radio-group`
**Provides:** Individual selectable option with accessible interaction and visual feedback

**Usage Example:**

```html
<mat-radio-group [(value)]="selectedSize">
  <!-- Basic radio button -->
  <mat-radio-button value="small">Small</mat-radio-button>

  <!-- Radio button with disabled state -->
  <mat-radio-button value="medium" [disabled]="true">
    Medium (Out of Stock)
  </mat-radio-button>

  <!-- Radio button with custom label position -->
  <mat-radio-button value="large" labelPosition="before">
    Large
  </mat-radio-button>

  <!-- Radio button with rich content -->
  <mat-radio-button value="xl">
    <div class="radio-content">
      <strong>Extra Large</strong>
      <p class="radio-description">Perfect for teams of 10+ people</p>
    </div>
  </mat-radio-button>

  <!-- Radio button with icons -->
  <mat-radio-button value="custom">
    <div class="radio-with-icon">
      <mat-icon>star</mat-icon>
      Custom Size
    </div>
  </mat-radio-button>
</mat-radio-group>
```

    const paymentMethod = this.getSelectedPaymentMethod();

    // Account type pricing (simplified - only counting numeric prices)

    // Shipping cost
    if (shipping) {
      total += shipping.price;

    // Payment method fee

## Component Relationships

### Data Flow

1. **Radio Group → Individual Buttons**: The `MatRadioGroup` manages the selection state and ensures only one radio button can be selected at a time
2. **Individual Buttons → Radio Group**: When a radio button is clicked, it notifies the radio group to update the selection state
3. **Radio Group → Form**: The radio group provides form control interface for integration with reactive forms and template-driven forms
4. **Selection Events**: Selection changes bubble up from individual buttons through the radio group to parent components

### Context Sharing

- **Selection Context**: `MatRadioGroup` maintains the current selection state and ensures mutual exclusivity among child radio buttons
- **Form Context**: Radio group provides form control interface and validation state management
- **Accessibility Context**: Radio group manages ARIA attributes, focus management, and keyboard navigation for all child buttons
- **Styling Context**: Radio group can provide consistent styling properties (color, label position) to all child radio buttons

### Event Handling

- **Selection Management**: Radio group coordinates selection changes and ensures only one option can be selected
- **Keyboard Navigation**: Arrow keys navigate between radio buttons, Space/Enter select options
- **Form Integration**: Selection changes trigger form validation and update form control values
- **Change Events**: Radio group emits change events that include both the selected radio button and its value

## Best Practices

### Usage Guidelines

- **Use for mutually exclusive choices**: Radio groups are specifically designed for selecting exactly one option from multiple choices
- **Limit options to 2-7 items**: For more than 7 options, consider using a select dropdown instead
- **Provide clear, concise labels**: Each radio button should have descriptive text that clearly indicates what the option represents
- **Set sensible defaults**: When possible, pre-select the most common or recommended option
- **Group related options logically**: Organize radio buttons in a logical order (alphabetical, by priority, or by frequency of use)
- **Use consistent layout**: Maintain consistent spacing and alignment across all radio groups in your application

### Accessibility

- **ARIA radiogroup**: Radio groups automatically provide proper ARIA radiogroup role and radio roles for screen readers
- **Keyboard navigation**: Supports arrow key navigation between options and Space/Enter for selection
- **Screen reader support**: Properly announces group labels, individual option labels, and selection state
- **Focus management**: Maintains proper focus indicators and manages focus when navigating between options
- **Label association**: Ensures proper association between radio buttons and their labels for accessibility

### Performance

- **Form optimization**: Use reactive forms with radio groups to leverage Angular's change detection optimizations
- **State management**: Implement efficient state management for large forms with multiple radio groups
- **Event handling**: Use appropriate change detection strategies to minimize unnecessary re-renders
- **Bundle optimization**: Import only the radio module to minimize bundle size
- **Memory management**: Properly unsubscribe from form value changes in component lifecycle
